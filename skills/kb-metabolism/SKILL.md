# kb-metabolism 全生命周期

薄壳技能：只负责路由，**所有规则与流程以命令输出为准**——任务提示由 kb 现场生成，随产品版本演进，勿在此复述协议知识。默认从当前目录向上找库；不在库内时加 `--vault <目录>`。

| 用户意图 | 做法 |
|---|---|
| 建库：把某目录管起来 | 问清管理范围（哪些子目录）、是否 git，然后 `kb init -y --managed <globs> [--git] --vault <目录>`；MCP 注册与语义检索按输出提示引导用户 |
| 分诊：给未分诊笔记定层 | `kb triage --emit`，严格按输出执行 |
| 每周消化：法医名单 + 审查 | `kb digest --emit`，严格按输出执行 |
| 提炼：高频资料变判断 | `kb chew --emit`，严格按输出执行（落盘命令在输出里，禁止用 kb_add 建 L0） |
| 过堂：用户在对话里下判决 | 名单条目编号列清 → **只勾选用户逐字点名的条目** → `kb execute <报告>` 汇报结果 |
| 体检/健康度 | `kb doctor`（或 MCP 工具 kb_stats） |

铁律兜底：永远不要替用户勾选处决名单、删除或移动笔记；判断与用途必须出自用户原话——AI 只提案，人是法官。

唯一例外：`kb key set`（语义检索密钥）必须用户亲手在终端执行——密钥不进对话历史。

支持 MCP prompts 的客户端（如 Claude Code）优先用斜杠命令：`/mcp__kb__triage`、`/mcp__kb__digest`、`/mcp__kb__chew`。
