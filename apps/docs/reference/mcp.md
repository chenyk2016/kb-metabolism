# MCP 工具

`kb serve` 以 stdio 暴露 MCP 检索门。门规随 initialize 注入接入方，不依赖客户端配置。

```bash
claude mcp add --scope user kb -- kb serve --vault ~/notes
```

## 五个工具

| 工具 | 职责 | 信号 |
|---|---|---|
| `kb_search` | 混合检索（字面三层 + 可选语义，RRF 融合）。自然语言直接查，无语法。 | 记 `kb_search` |
| `kb_read` | 读笔记全文（路径相对 vault 根，限管理范围内） | 记 `kb_read`——**续命 90 天** |
| `kb_add` | 捕捉新笔记。**入口税**：无 `use_when` 只能进 inbox；**查重税**：疑似同主题时不写入、返回候选，确认新主题才 `force=true` | 记 `kb_add` |
| `kb_cite` | 声明产出实际使用了哪些笔记 | 记 `kb_cite`——**续命 180 天** |
| `kb_stats` | 库健康度：层级分布、L0 容量、孤儿、门流量 | 无 |

## 门规（注入给 agent 的规则）

1. 查个人积累的知识先 `kb_search` 再回答，读笔记用 `kb_read`——**不要绕过门直接读库内文件**（绕过 = 不留信号 = 给全部笔记发免死金牌）
2. 存知识用 `kb_add`；返回"疑似同主题"时先 `kb_read` 候选，同主题直接编辑该文件补充（一个主题一篇笔记），确认新主题才带 `force` 重试
3. 编辑已有笔记直接改文件，改完可跑 `kb index` 刷新
4. 产出中实际用到某笔记就 `kb_cite`——只 cite 真正用上的，不要客套性引用
5. **永远不要替用户勾选处决名单、删除或移动笔记**——AI 只提案，人是法官

## 提醒的搭便车

距上次消化超一周时，`kb_search` / `kb_stats` 的结果尾部会带一行 ⚠️ 提醒并请 agent 转告用户——代谢节律寄生在最高频的接触点上，不依赖任何人的自觉。
