# 蓝图

> 完整版在仓库 `docs/blueprint.md`；本页是公开摘要。

## 终态一句话

**agent 时代个人判断力的主权层。** 最终交付物是那份 ≤100 条、可移植、每条带使用证据链的 L0——其余一切（L1、inbox、检索、法医、墓地）都是生产这 100 条的供应链。

## 五层一内核

```
┌─ 云/团队层 ──────── 同步、团队 vault、归属人判决
├─ 分发层 ─────────── npm / Obsidian 插件 / 文档站（本站）
├─ 界面层 ─────────── kb ui 管理台（判决台 + 体检室）
├─ 接口层 ─────────── CLI ─ MCP ─ HTTP API（带版本的正式产品面）
├─ 内核 ───────────── 协议 + 引擎（索引/法医/体检/消化），零 LLM 完整可用
└─ 真相层 ─────────── 你的 Markdown + git + 信号 JSONL（属于你，不属于产品）
```

## 永久不变量

1. 文件 + git 是真相，一切索引可丢弃重建
2. 访问日志是唯一不可再生数据（JSONL 随 vault）
3. 检索必须走门；管理性浏览不续命
4. AI 提案、人判决、git 兜底可反悔；删除永远不给 AI
5. 零 LLM 时内核完整；判断力是插件
6. vault 自包含：拷走目录 = 带走整个系统

## 为什么押"可治理的遗忘"

模型厂商的原生记忆只会越来越好，"帮你记住"没有护城河；但黑盒、云端、不可审计、不可迁移是它们的结构性宿命。**可解释、可否决、可反悔、跟人走**的记忆治理，是留给文件型方案的空地——这不是时间窗口，是站位差异。
