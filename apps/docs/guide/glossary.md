# 概念表

本产品的"黑话"词汇表。每个概念都对应一个真实机制，不是修辞。

## 分层

| 概念 | 含义 |
|---|---|
| **L0 核心** | 人读的判断层。≤100 条硬上限，每条 = 一句可复述的判断 + 何时会再用 + 证据链。满员必须先挤掉一条才能收新。 |
| **L1 资料** | AI 读的资料层。能被检索到即可，人不通读。零信号超过衰减窗口 → 处决名单。 |
| **inbox** | 暂存层。默认 30 天到期，到期前要么升级要么进名单。写不出用途的内容只能进这里。 |
| **墓地（_graveyard/）** | 被掩埋的笔记。`git mv` 移入，随时可还魂；墓地内容随时可真删（真删归人手动）。 |

## 进出

| 概念 | 含义 |
|---|---|
| **入口税** | 进 L0/L1 必须写出 `kb_use_when`（什么时候会再用到）。答不上来 = 进 inbox 等它自己证明价值。 |
| **查重税** | 新增内容与已有笔记主题重合过半时不写入，先看候选再决定。一个主题一篇笔记，宁可长文不要碎片。 |
| **分诊（triage）** | 给未定层的笔记定层。可交互、可 LLM 提案、可 emit 给 agent。 |
| **处决 / 赦免 / 过堂** | 法医名单的人工判决环节。处决 = git mv 进墓地；赦免 = 保留，但除非获得使用信号，下周仍会上榜。 |

## 信号

| 概念 | 含义 |
|---|---|
| **门** | 唯一的检索入口（CLI / MCP / 管理台共用）。每次走门都追加一条信号到 `access.log.jsonl`。 |
| **续命** | 使用信号让笔记免于上榜：被读取免死 90 天，被引用免死 180 天。 |
| **kb_cite（引用）** | agent 声明"我的产出实际用了这篇笔记"。最高等级存活证据。 |
| **kb_inject / kb_ui** | hooks 自动注入、管理台浏览产生的观察信号——**法医不认，不续命**。机器注入 ≠ 人在使用；管理界面翻库 ≠ 使用。 |
| **法医（coroner）** | 纯数据的验尸程序：按信号、反链、修改时间列出处决候选。只提案，永不删除。 |
| **吸收** | 知识被用进真实产出。`kb_cite` + 创作目录（`outputDirs`）反链是两条吸收通道；`kb doctor` 报告吸收率。 |

## 消化

| 概念 | 含义 |
|---|---|
| **消化（chew）** | 把反复被读的 L1 资料提炼成一句 L0 判断。AI 是消化酶不是胃——拆解可以自动，**判断必须由人亲口说出**。 |
| **kb_digested** | 源资料被消化后的标记：营养已进入 L0，之后可自然衰亡，人审时可放心处决。 |
| **体检（doctor）** | 不依赖信号的健康诊断：年龄分层、孤儿率、吸收率、语义层状态。新库第一分钟即可用。 |
