# 日常使用

## 重要：几乎不用"用"它

这个系统的设计哲学是**尽量不被使用**——它是器官，不是要天天打开的工具。

- **笔记照常写**：在你自己的编辑器里，想怎么写怎么写。直接改文件即可，mtime 和 git 天然记账。
- **查东西问 agent**：接了门的 agent（见[接入 agent](/guide/agents)）检索自动走 `kb_search`，每次查询都在给命中的笔记续命。装了 hooks 的话连问都不用特意问——每条对话自动带上相关笔记。
- **存东西对 agent 说**："存进知识库，用途是 XX"。`kb_add` 会自动执行入口税和查重税。

## 不知道该干什么的时候

直接敲：

```bash
kb
```

它会看一眼你的库，按优先级告诉你此刻该做的事（未分诊几条、名单几条待审、几篇值得消化），或者告诉你什么都不用做。

## 手动捕捉（带税）

```bash
kb add "SQLite 中文检索" --use-when "下次选全文检索引擎时"   # → L1
echo "一次性调研内容" | kb add "临时发现"                    # → inbox，30 天过期
```

写不出 `--use-when` 的内容只能进 inbox——这不是限制，是保护：捕捉的摩擦太低，你就会存垃圾。

## 门自己会提醒你

不需要记住任何节律。距上次消化超过一周时，检索结果和统计的尾部会带一行 ⚠️ 提醒；接了门的 agent 也会转告你。到那时再花五分钟即可：[每周节律](/guide/weekly)。
