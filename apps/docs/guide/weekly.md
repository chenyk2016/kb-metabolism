# 每周节律

每周五分钟，三个动作。忘了跑也没关系——门自己会提醒。

## 1. 消化仪式：`kb digest`

```bash
kb digest
```

一条命令做完：重建索引 → **法医验尸**（零信号笔记列成处决名单）→ 体检留档 → 消化名单（高频被读的资料）。

法医是纯数据的，它只提案、永不删除。一篇笔记上榜需要同时满足（详见[协议规范](/protocol/spec#法医判定规则)）：

- 访问日志超过衰减窗口零读取（默认 90 天）
- 超过引用窗口零引用（默认 180 天）
- 超过注入窗口零注入（默认 30 天）
- 活反链数为 0（全 vault 扫描；只有自身活着的来源发出的链接才算担保，创作目录的引用永远算）
- 内容超过衰减窗口未变更（重命名与批量提交不算触碰）
- inbox 层：`kb_expires` 过期即候选

## 2. 过堂：`kb review`

```bash
kb review
```

逐条 y/n 判决：**y = 处决，n = 赦免**。判完自动执行——批准的条目 `git mv` 到 `_graveyard/` 并提交，永远可反悔。

赦免不是永久豁免：除非它获得使用信号，下周还会上榜。这是刻意的——赦免的成本必须存在，否则名单会变成橡皮图章。

喜欢图形界面就用 [`kb ui` 的过堂视图](/guide/console)：盖章判决，一屏搞定。

## 3. 消化：`kb chew`（偶尔）

排泄只是手段，**消化才是目的**——人们要的不是干净的库，是"存的东西变成判断力"。

```bash
kb chew
```

近 90 天被反复读取（≥2 次）的 L1 资料有营养，值得提炼。流程里 AI 是消化酶不是胃：它可以拆解出候选判断、追问"当初的用途还成立吗"，但**最终判断必须由你亲口说出**——把反思外包给 AI，正是第二大脑失败的病根。

产出的 L0 = 一句可复述的判断 + 用途 + 证据链。源资料自动标记 `kb_digested`：营养已转移，之后无人再读可自然衰亡——消化加速排泄，库里不再有中间态囤积。
