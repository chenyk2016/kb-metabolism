# 管理台（kb ui）

```bash
kb ui              # 默认 127.0.0.1:7317，自动开浏览器
kb ui --port 8000 --no-open
```

本机起一个**判决台 + 体检室**。它不是又一个笔记应用——定位是每周五分钟来一次的地方，不是每天打开的编辑器。

## 七个视图

按你的三种角色分组：

| 组 | 视图 | 做什么 |
|---|---|---|
| **看** | 总览 | 体检指标（年龄分层、L0 余量、门流量、吸收率）+ 此刻该做的事 |
| | 笔记 | 分层浏览、混合检索、详情抽屉（正文/反链/信号史）、捕捉（带查重税）|
| | 信号 | access.log.jsonl 流水，按工具过滤 |
| **判** | 过堂 | 处决名单逐条盖章（处决/赦免）→ 执行掩埋 |
| | 分诊 | 未分诊队列逐条定层，入口税强制 |
| | 消化 | 左侧源资料、右侧亲口写判断 → 落成 L0 |
| **管** | 设置 | config 表单、报告历史、墓地还魂、重建索引 |

## 三条纪律（刻在管道里，不是约定）

1. **界面浏览不续命**。管理台里的浏览与检索记 `kb_ui` 观察信号——法医不认。在界面里翻一遍全库 ≠ 使用，否则法医形同虚设。想让一篇笔记活下去：去用它。
2. **没有删除按钮**。全站唯一能让笔记消失的路径仍是 勾选名单 → 执行（git mv 可反悔）；墓地里可一键还魂。
3. **没有编辑器**。改内容回你自己的编辑器——文件永远是唯一真相，管理台只是派生视图。

## 安全边界

服务只绑 `127.0.0.1`；所有写操作校验来源（防浏览器内恶意页面攻击本地端口）；路径参数严格限制在库内且永远不允许触碰 `.kb/`（secrets 所在地）；API key 永不经过任何接口——配置它只有一条路：终端里 `kb key set`。
