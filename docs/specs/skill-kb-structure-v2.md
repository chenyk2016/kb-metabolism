# kb-structure v2：目录结构 + 命名规范的独立整理技能

> 状态：待确认 | 日期：2026-07-10
> 前提：phase-9 kb_id 已解除"移动=信号清零"，移动/改名成为安全廉价动作。
> 动机：对抗式审查结论——守门（prose）拦不住不走门的自动化写入；缺的是可机检的体检与廉价的矫正循环。

## 第一性重构：从"守门员"到"整理循环"

漂移不可避免（写入方太多、多数不经过任何技能）。系统健康不取决于"永不违规"，
而取决于**违规被发现的延迟**和**矫正的成本**。因此技能定位改为三职能：

| 职能 | 载体 | 时机 |
|---|---|---|
| 守门 | prose 决策程序（v1 已有） | 用户开口的存放/建目录/移动时 |
| **体检** | **scripts/audit.mjs（新，零依赖 node）** | 用户说"检查/体检/整理知识库结构"时；建议每周随 digest 跑 |
| **整理** | 审计报告 → 提案清单 → 人批 → 执行 | 体检发现违规后 |

可机检的规则交给脚本判（确定、可复跑、不占上下文），不可机检的语义判断留给口诀。

## 命名规范（第一性推导，新增进宪法）

命名的消费者决定规则，每条都有推导来源：

| 规则 | 推导来源 |
|---|---|
| **basename 全库唯一**（不含扩展名的文件名不得重复） | kb 反链按 basename 解析（indexer `byBasename`）——同名笔记会让反链记账错乱，这是实现机制推出的硬约束 |
| 捕捉笔记 `YYYY-MM-DD-主题.md` | 时间是捕捉时唯一零摩擦的轴；日期前缀天然排序、天然唯一化 |
| 目录 `NN-名称`（两位数字前缀） | Johnny.Decimal：锁定侧边栏排序 + 可口头寻址（"放到 10 下面"） |
| 中文允许；禁止空格与 `\/:*?"<>|`；连接用 `-` | 人可读优先（中文生态）；跨平台路径安全；与 kb `safeFilename` 一致 |
| assets：每个顶层区一个 `assets/`，文件名用内容标识（token/hash） | 图片生命周期绑定笔记但引用要短；区级粒度平衡"同迁成本"与"路径深度"；内容标识天然免撞名 |
| 改名 = 高危动作（basename 变 → wikilink 解析受影响），必须附全库引用改写 | 反链机制的另一面 |

## scripts/audit.mjs：结构体检六检

输入 `--vault <root>`（读库内 `90-system/目录规范.md` 的机读参数节 + `.kb/config.json`），输出违规报告（stdout markdown + `--json`）：

1. **顶层膨胀**：顶层条目数 > 宪法上限（默认 10，排除 `_`/`.` 前缀）
2. **深度超限**：> 2 层的目录路径（列最深五条 + 总数）
3. **命名违规**：不合命名规范的文件/目录（空格、非法字符、无数字前缀的顶层目录等）
4. **basename 撞名**：全库同名 md（反链错乱源）
5. **孤儿 assets**：无任何笔记引用的图片；及笔记引用了但不存在的图片（断链）
6. **内容副本**：managed 范围内容 hash 相同的文件对（kb_id 撞号重发只治身份，不治冗余）

报告按"修复动作"分组直接给出提案（mv/rename/删除候选），每条附理由——**报告即提案清单，人勾选后技能执行**。

## 整理执行流程（技能 prose 部分）

```
audit → 提案清单 → 人批准 → 逐条执行：
  移动:   git mv → 检查正文 assets 相对引用同迁
  改名:   git mv → grep 全库 [[旧名 与 ](旧路径 → 改写引用 → 报告改写数
  删除类: 永不执行，转交 kb 处决名单流程（AI 提案人判决）
→ kb index --vault <root>（kb_id 认亲）→ 分批 commit
```

## 宪法同步修订（模板 + 用户库实例）

1. 新增"命名规范"节（上表规则 + 机读参数：顶层上限、深度上限、日期前缀正则）
2. **删除**"归档动作顺手写元数据"句——kb 协议无归档字段，tier 降级语义错误（对抗审查 bug #3）
3. 用户库宪法：迁移蓝图与处决候选**移出**（挪到 agent-space 调研文档），宪法只留不变量 + 最小状态（对抗审查 #2）

## 不做什么

- 不改 kb CLI/协议（独立技能，审计脚本自带；将来若 kb doctor 想吸收六检，脚本可平移）
- 不自动执行任何移动/改名/删除——报告即提案，人是法官
- 不处理 save-to-obsidian 落点冲突（另行收口，本 spec 只声明边界）
- 不做定时调度（用户可自行挂 cron 或随周消化跑）

## 验收标准

1. 在真实库跑 audit：能查出已知违规（顶层 32 项、`10-projects/99-存档` 5 层深、今日 16 张图的引用完整性）且零误报致命项
2. 撞名检测：人为制造同名 md 被抓出
3. 改名流程：改一篇有反链笔记的名，全库引用被改写，`kb index` 后反链数不变
4. 三个触发 eval：存文章（守门）、建顶层目录（拦截）、"帮我体检下知识库结构"（审计）
