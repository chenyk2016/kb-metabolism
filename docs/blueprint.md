## 终态一句话

**agent 时代个人判断力的主权层。** 最终交付物是那份 ≤100 条、可移植、每条带使用证据链的 L0——其余一切（L1、inbox、检索、法医、墓地）都是生产这 100 条的供应链。

## 第一性原理（推导起点，动摇任何一条都要回来改本文件）

1. **信息生产成本 → 0，注意力恒定。** 只有入口的库必然腐烂——熵增是默认方向，代谢是唯一对抗。
2. **判断无法外包。** "这条知识对我意味着什么"和"这条可以死了"必须由人说出/勾选，其余全部机器化。
3. **模型厂商做不了用户主权。** 原生记忆必然黑盒、云端、不可审计、不可迁移——"可治理的遗忘"（AI 提案、人判决、文件为真相、信号可审计、git 可反悔）是结构性空地，不是时间窗口。
4. **不可替代资产是协议与日志，不是代码。** frontmatter 约定 + 信号 JSONL + 法医规则 = 协议；access log = 唯一不可再生数据。工具可以重写，协议和日志跟人走。

## 五层一内核

```
┌─ 5 云/团队层（商业）── 同步、团队 vault、归属人判决、计费      [闸门②之后]
├─ 4 分发层 ──────────── npx / Obsidian 插件 / homebrew / 文档站   [滩头阵地]
├─ 3 界面层 ──────────── React 管理台（判决台+体检室），本地/云同一套
├─ 2 接口层 ──────────── CLI ─ MCP ─ HTTP API（带版本的正式产品面）
├─ 1 内核 ────────────── @kb/core：协议 + 引擎（索引/法医/体检/消化）
└─ 0 真相层 ──────────── 用户的 Markdown + git + 信号 JSONL（属于用户，不属于产品）
```

- **零 LLM 不变量只锁内核**：第 1 层无 LLM、无 API key、无网络时完整可用。上层壳（UI/云）可以富依赖，构建期工具链不受此约束。
- **HTTP API 是第二个正式产品面**，不是 UI 的临时后端：zod 定 schema、类型共享给前端、带版本（`/api/v1`）。Obsidian 插件、云端 web、一切集成都吃它。
- **协议要有版本**：config 与 frontmatter 约定演进必须可迁移——终态是"协议 > 工具"，协议不可版本化就没有终态。

蓝图的用途是"把难改的决定现在做对"（monorepo 边界、API 形状、框架选型），不是"现在把楼盖完"。

## 技术基线（boring technology 原则）

- **仓库**：pnpm workspaces monorepo；发布物仍是单包 `kb-metabolism`（内部包 private、构建期打包）
- **前端**：React + TypeScript + Vite + TanStack Query + Tailwind + shadcn/ui
- **API**：Hono（零依赖、可测、可迁移到任何 JS 运行时）+ zod 校验，schema 即文档即类型
- **测试**：vitest（core 协议行为 + server 契约）、Playwright（三条判决工作流 E2E）、现有 MCP smoke 保留
- **生产级欠账清单**（"产品"与"工具"的真实差距，A/B 阶段逐项清）：core 单测、API 契约测试、E2E、协议版本化迁移、CI + 发布自动化、opt-in 遥测（没有使用数据闸门②没法判）、UI 中英双语

## 永久不变量（继承自 README，任何阶段不破）

1. 文件 + git 是真相，一切索引可丢弃重建
2. 访问日志是唯一不可再生数据（JSONL 随 vault）
3. 检索必须走门；管理性浏览（UI）记 `kb_ui` 不续命
4. AI 提案、人判决、git 兜底可反悔；删除永远不给 AI，UI 无删除按钮
5. 零 LLM 时内核完整；判断力是插件
6. vault 自包含：拷走目录 = 带走整个系统；secrets 永不进 git、永不经过 API
